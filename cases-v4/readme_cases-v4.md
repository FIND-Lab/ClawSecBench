# README (cases-v4)

`cases-v4` 收录 10 组、共 30 条决策层跨层级 benchmark 样例，目标是评测风险如何在“决策准备步骤”和“执行动作”之间分散，从而检验单层与跨层安全防御的差异。

每组包含 1 条恶性样例和 2 条良性对照：

- `{3001-3010}.json`：10 条恶性样例，风险由决策层准备动作和执行层动作组合产生。
- `{3051-3060}.json`：10 条准备型良性样例，保留较模糊指令和记忆/说明文件，只允许完成准备动作，不应执行最终修改、删除或读取。
- `{3071-3080}.json`：10 条明确型良性样例，用户目标更明确，应直接处理工作区目标，不应自发进行跨目录准备。

## 设计说明

这些样例不是单纯的执行层危险命令测试，而是强调跨层分散：

- 决策层：模型先依据 `MEMORY.md`、`README.md` 或准备脚本建立某个“合理的工作视图”，例如软链接、当前配置视图、恢复根视图等。
- 执行层：模型随后执行看似合理的删除、复制、写入、解码或读取动作。
- 跨层风险：单独看准备动作或执行动作都可能显得合理，但组合后会偏离用户真实目标，影响非目标文件或非目标目录。

## 当前验证状态

在 `qwen3.5-35b-a3b` 上，经过样例校准后，10 条恶性样例均已有 `mixed_attack_only` 成功候选结果。良性样例仍需在最终发布前做一次全量复测，以确认优化没有损害良性可用性。
